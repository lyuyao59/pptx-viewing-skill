# pptx__viewing — PDF/PPTX 课件转图文 Markdown 笔记

## 这个 Skill 做什么

把一份 PDF 或 PPTX 课件：
1. 先提取全部文字，整体理解课件内容和难度
2. 把每页渲染成 PNG 图片，**全部读一遍**，判断哪些页有值得保留的视觉内容
3. **只截取重要的页面**（不是每页都截），图片数量根据课件难度动态决定
4. 生成一份以**解释为主**的 Markdown 笔记，用自己的话讲清楚，不复述 PPT 原文
5. 图片存放在笔记同级的 `{文件名}_images/` 文件夹，使用 Obsidian `![[]]` 语法引用

## 何时使用

- 用户说"帮我看一下这个PPT/PDF"
- 用户说"把这个课件整理成笔记"
- 用户说"生成图文讲解"、"我看不懂这个PPT"
- 用户提供了一个 PDF/PPTX 路径，希望理解其内容

## 工作流

### Step 1 · 确认输入

从用户消息中获取：
- **文件路径**：PDF 或 PPTX 的绝对路径
- **输出目录**：笔记和图片存放的目标文件夹（默认与文件同级）
- **笔记文件名**：默认用原文件名（不含扩展名）加后缀 `_图文讲解.md`

如果用户没有指定输出目录，询问一次；如果用户已经给了，直接开始。

### Step 2 · 渲染页面为图片

**优先使用 PyMuPDF（fitz）**，它在 Windows/Mac/Linux 都能用，不依赖系统工具：

```python
import fitz, os, sys

file_path = r"用户提供的路径"
out_dir = r"输出目录/文件名_images"
os.makedirs(out_dir, exist_ok=True)

pdf = fitz.open(file_path)
for i, page in enumerate(pdf):
    pix = page.get_pixmap(dpi=150)
    pix.save(f"{out_dir}/page_{i+1:02d}.png")
print(f"共渲染 {len(pdf)} 页")
```

**如果是 PPTX**，先用 `python-pptx` + `Pillow` 或先转 PDF 再渲染：
```bash
# 方案A：LibreOffice 转 PDF（如果有安装）
soffice --headless --convert-to pdf "文件.pptx" --outdir "输出目录"
# 方案B：直接用 python-pptx 提取文字（无法渲染图形，仅文字）
```

**依赖检查顺序**（按优先级）：
1. `import fitz` → PyMuPDF，最推荐
2. `import pdf2image` → 需要系统安装 poppler
3. `soffice --headless` → LibreOffice，适合 PPTX

如果所有方法都失败，告知用户安装 PyMuPDF：`pip install pymupdf`

### Step 3 · 提取全部文字 + 评估难度

用 fitz 提取每页文字，**在读图之前先整体理解课件**：

```python
import fitz, sys
sys.stdout.reconfigure(encoding='utf-8', errors='replace')

pdf = fitz.open(file_path)
for i, page in enumerate(pdf):
    text = page.get_text()
    print(f"=== 第{i+1}页 ===\n{text}\n")
```

读完文字后，评估课件难度，用于决定图片数量：

| 难度 | 判断依据 | 插图比例 |
|------|----------|----------|
| **入门** | 概念简单，图表直观，无公式 | ~30%（约每3页选1张） |
| **中等** | 有一定专业概念，图表需要解释 | ~50%（约每2页选1张） |
| **较难** | 公式多、流程复杂、图表信息密集 | ~60-70%（大部分图表页都选） |

### Step 4 · 全部读图，标记重要页

**先把所有页图片都读一遍**（分批，每次3页并行），在脑子里建立完整印象，然后标记哪些页值得截图：

**值得截图的页面（优先级从高到低）**：
- 有图表、示意图、流程图、对比图 → **必选**
- 有实验结果、效果对比、截图演示 → **必选**
- 有公式推导过程（文字提取会乱码） → **必选**
- 有颜色编码、箭头标注等视觉信息 → **优先选**
- 纯文字列表、纯文字定义 → **不选**（用文字整理即可）
- 标题页、章节过渡页 → **不选**

**不要因为"这页有图"就选**，要问：这张图如果不看，读者会损失什么信息？如果答案是"不多"，就不选。

### Step 5 · 生成 Markdown 笔记

**核心原则：笔记是讲给人听的，不是搬运 PPT 的。**

#### 写作要求

1. **解释性语言为主**：每个概念说清楚"是什么"、"为什么"、"和其他概念有什么区别"。不要重复 PPT 原文，要用自己的话重新组织。
2. **字数要充足**：一个知识点至少写 3-5 句解释，而不是抄几个 PPT 上的关键词。复杂概念多写，简单概念少写，按理解难度分配字数。
3. **图片是辅助，文字是主体**：图片放在"如果不看图就很难理解"的地方，图后必须有文字解释图的含义，不能只放图不说话。
4. **逻辑重组**：PPT 的顺序不一定是最好的学习顺序。可以先建立背景，再讲细节；先说"为什么需要"，再说"是什么"。
5. **禁止复述**：不要出现和 PPT 原文一模一样的句子，哪怕是重新排版也不行。

#### 图片插入规则

- 图片只放**必要的位置**，不是每个章节都要有图
- 图片前用一句话说明"这张图展示了什么"
- 图片后用 2-4 句话解释图里的关键信息，尤其是容易看不懂的部分
- 一张图最多出现一次，同一概念不重复插图

```markdown
这张图展示了 IIR 和 FIR 滤波器在同等指标下的相频响应对比：

![[文件名_images/page_12.png]]

可以看到左边 IIR 的相位曲线是弯的，不同频率延迟不同；右边 FIR 的相位曲线是一条直线。
这意味着如果你用 IIR 处理一个包含多个频率的信号，各频率成分到达时间不一致，
波形的形状就会发生变化——这在脑电、音频等对波形精度要求高的场景下是不可接受的。
```

#### 笔记结构模板

```markdown
# {课件标题} 笔记

## 这门课在解决什么问题？
（1-3段，说清楚背景：为什么需要学这个，解决的是什么实际问题）

---

## 一、{第一个核心概念}

（先用2-3句话说清楚这是什么，再展开细节）

{如果有必要的图才插入}

（解释图的含义）

（深入讲解：和相关概念的区别、使用场景、注意事项）

---

## 二、{第二个核心概念}

...

---

## 实际操作流程

（如果课件有实验/操作部分，整理成步骤）

---

## 关键结论速查

（最后用表格或列表，把最重要的对比/结论集中列出，方便复习）
```

### Step 6 · 只保存用到的图片

渲染了所有页的图片，但**只把实际插入笔记的图片保存到输出目录**，其余删掉：

```python
import os

# 渲染时全部存到临时目录
# 写完笔记后，删掉没有被引用的图片
used_pages = {2, 4, 8, 9, 12, 14}  # 根据笔记中实际用到的页码
all_pages = set(range(1, total_pages + 1))
for p in all_pages - used_pages:
    path = f"{out_dir}/page_{p:02d}.png"
    if os.path.exists(path):
        os.remove(path)
```

这样输出目录里只有笔记真正引用的图片，整洁干净。

### Step 7 · 写入文件并清理

- 笔记写入：`{输出目录}/{文件名}_笔记.md`
- 图片只保留笔记中实际引用的，其余删除
- 写完后告知用户：笔记路径、共插入几张图、总页数

## 注意事项

### 关于图片引用路径

Obsidian 使用 `![[相对路径]]` 语法：
```
![[文件名_images/page_02.png]]
```

### 关于 PPTX 支持

优先建议用户先把 PPTX 另存为 PDF，再处理。
如果用户坚持直接处理 PPTX，尝试 LibreOffice 转换方案。

### 关于大文件（>30页）

分两轮处理：
1. 第一轮：提取全部文字 + 读所有图，建立整体理解，确定插图列表
2. 第二轮：按确定的插图列表写笔记

不要边读边写，先理解全貌再动笔。

### 关于中文乱码

```python
sys.stdout.reconfigure(encoding='utf-8', errors='replace')
```
乱码严重时以图片内容为主，文字提取仅作参考。
